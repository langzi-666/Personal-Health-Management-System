## 2. 技术要点与方案

### 2.1 实现的功能特性

**后端架构设计**
```
采用标准的分层架构：
- Controller层：处理HTTP请求
- Service层：业务逻辑处理
- Mapper层：数据库操作（MyBatis Plus）
- Entity层：数据实体
- DTO层：数据传输对象
- Utils层：工具类
- Config层：配置类
```

**前端架构设计**
```
采用现代的Vue 3 Composition API：
- 组件化开发
- 路由管理
- 状态管理（Pinia）
- API请求层（Axios）
- Element Plus UI框架
```

**数据库设计**
```
特点：
- 11个核心数据表
- 完整的外键关系
- 合理的索引优化
- 统一的时间戳字段
- UTF8MB4字符集支持
```

### 2.2 遇到的技术问题与解决方案

**问题 1**：Spring Boot与JWT集成
- **原因**：需要正确配置JWT的密钥和过期时间
- **解决方案**：在application.yml中配置，使用@Value注解注入到JwtUtil
- **花费时间**：0.5小时

**问题 2**：Vue 3 Composition API的学习曲线
- **原因**：Composition API的语法与Options API不同
- **解决方案**：使用<script setup>语法简化代码
- **花费时间**：0.5小时

**问题 3**：前后端API通信配置
- **原因**：需要正确配置跨域和API代理
- **解决方案**：在Vite中配置proxy，在Spring中配置CORS
- **花费时间**：0.5小时
