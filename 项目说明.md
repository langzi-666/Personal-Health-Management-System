# 个人健康管理系统 - 项目说明

## 📋 项目概览

**项目名称**：个人健康管理系统 (Personal Health Management System)  
**项目类型**：大学生毕业设计项目  
**技术栈**：Vue.js 3.x + Spring Boot + MySQL 8.0  
**开发周期**：8周（MVP版本）  
**数据库配置**：
```
主机：localhost
端口：3306
用户名：root
密码：123456
数据库名：health_management_system
```

### 🗄️ MySQL数据库设计.md
**文件用途**：定义数据库表结构和设计规范  
**主要内容**：
- 数据库配置信息
- 11张数据表的详细设计
  - users（用户表）
  - weight_records（体重记录）
  - blood_pressure_records（血压记录）
  - blood_glucose_records（血糖记录）
  - exercise_records（运动记录）
  - sleep_records（睡眠记录）
  - diet_records（饮食记录）
  - mental_health_records（心理健康记录）
  - notifications（通知）
  - reminder_rules（提醒规则）
  - alert_rules（告警规则）
- 表字段说明和建表 SQL
- 数据关系图
- 索引策略和性能优化建议

**何时查看**：需要执行数据库初始化时；编写数据库相关代码时

**使用方法**：
1. 创建数据库：`CREATE DATABASE health_management_system CHARACTER SET utf8mb4;`
2. 选择数据库：`USE health_management_system;`
3. 复制所有 SQL 语句执行以创建所有表

## 🎓 毕业设计提示

作为大学生毕业设计，注意以下几点：

1. **论文撰写**：
   - 详细记录开发过程和技术方案
   - 使用 开发总结_第X周.md 中的内容作为论文素材
   - 重点突出技术创新点和解决的问题

2. **演示准备**：
   - 准备演示脚本和操作流程
   - 提前测试所有功能，确保演示顺利
   - 准备备用方案（如系统无法启动）

3. **代码规范**：
   - 添加详细的代码注释，特别是复杂逻辑
   - 遵循命名规范和项目结构
   - 使用 Git 进行版本控制，提供完整的开发记录

4. **性能考虑**：
   - 虽然是毕业设计，但代码质量要达到生产级别
   - 优化数据库查询，避免 N+1 问题
   - 优化前端性能，图表加载和响应要快
